# Spring自动装配 Autowire

## 1. 传统的依赖注入方式
在不使用自动装配时，如果我们需要完成依赖注入，需要手动对 `property` 参数进行显式配置：

```xml
<bean name="student" class="com.test.bean.Student">
    <property name="teacher" ref="teacher"/>
</bean>
```

## 2. 开启自动装配 (基于 Setter 方法)
为了简化配置，我们可以开启自动装配。自动装配允许 IoC 容器自行寻找需要填入的值并完成注入，前提是我们需要在 Bean 中提供好对应的 `set` 方法。

开启自动装配只需在 `<bean>` 标签中添加 `autowire` 属性：

```xml
<bean name="student" class="com.test.bean.Student" autowire="byType"/>
```

**`autowire` 属性的两个常用值：**
* **`byName`**：根据Student类里的set方法 提取出对应名字 去找xml里注册的Bean。
* **`byType`**：根据属性的类型去寻找合适的 Bean 进行自动装配。

*注：配置完成后，IDE（如 IDEA）中的 set 方法旁边通常会出现一个自动装配的提示图标，表明配置生效，此时不再需要显式指定 `property`。*

## 3. 构造方法的自动装配
自动装配同样支持通过构造方法完成的依赖注入。此时需要将 `autowire` 属性的值修改为 `constructor`：

```xml
<bean name="student" class="com.test.bean.Student" autowire="constructor"/>
```

* **实现要求**：只需要在类中提供一个对应参数的构造方法即可。
* **匹配规则**：这种情况下，Spring 默认也是通过 `byType`（类型）去寻找并完成自动注入的。

## 4. 解决自动装配的歧义性问题
自动装配虽然方便，但过于机械。例如，当 `autowire` 规则设定为 `byType` 时，如果 IoC 容器中存在多个同类型的候选 Bean，Spring 就会因为无法确定注入哪一个而报错（歧义性问题）。

针对这种情况，有以下两种解决方案：

### 4.1 关闭特定 Bean 的自动装配候选资格
如果我们希望某个特定的 Bean（例如 `ProgramTeacher`）在任何情况下都不参与到自动装配中，可以使用 `autowire-candidate="false"` 属性将其排除：

```xml
<bean name="teacher" class="com.test.bean.ArtTeacher"/>
<bean name="teacher2" class="com.test.bean.ProgramTeacher" autowire-candidate="false"/>
<bean name="student" class="com.test.bean.Student" autowire="byType"/>
```
*原理*：当 `teacher2` 的 `autowire-candidate` 设定为 `false` 时，它将不再作为自动装配的候选 Bean。此时候选 Bean 只剩下唯一的 `teacher` (ArtTeacher)，歧义消除，程序正常运行。

### 4.2 设定首选 (Primary) Bean
除了排除候选者，我们也可以使用 `primary="true"` 属性来指定一个主要的 Bean。当出现类型匹配歧义时，Spring 会优先选择设定了 `primary="true"` 的 Bean：

```xml
<bean name="teacher" class="com.test.bean.ArtTeacher" primary="true"/>
<bean name="teacher2" class="com.test.bean.ProgramTeacher"/>
<bean name="student" class="com.test.bean.Student" autowire="byType"/>
```
*原理*：虽然存在多个类型匹配的候选者，但由于 `ArtTeacher` 被标记为了首选 Bean，Spring 会自动将其注入到 `Student` 中，程序得以正常运行。
