# 使用注解开发

前面我们已经完成了大部分的配置文件学习，但是我们发现，使用配置文件进行配置，貌似有点太累了吧？可以想象一下，如果我们的项目非常庞大，整个配置文件将会充满Bean配置，并且会继续庞大下去，能否有一种更加高效的方法能够省去配置呢？还记得我们在JavaWeb阶段用到的非常方便东西吗？没错，就是注解。

既然现在要使用注解来进行开发，那么我们就删掉之前的xml配置文件吧，我们来看看使用注解能有多方便。

## 1. 注解开发基础配置

### 1.1 切换注解上下文实现
现在我们使用 `AnnotationConfigApplicationContext` 作为上下文实现，它是注解配置的。

```java
ApplicationContext context = new AnnotationConfigApplicationContext();
```

### 1.2 创建配置类
既然现在采用注解，我们就需要使用类来编写配置文件。在之前，我们如果要编写一个配置的话，需要编写冗长的 XML 文件，现在我们只需要创建一个配置类就可以了：

```java
@Configuration
public class MainConfiguration {
}
```

这两者是等价的，同样的，在一开始会提示我们没有配置上下文：

![image-20221123175555433](https://s2.loli.net/2022/11/23/YNl5nVkfJriogdD.png)

这里按照要求配置一下就可以，同上，这个只是会影响IDEA的代码提示，不会影响程序运行。

### 1.3 指定默认配置类
我们可以为 `AnnotationConfigApplicationContext` 指定一个默认的配置类（这个构造方法可以接收多个配置类，更准确的说是多个组件）：

```java
ApplicationContext context = new AnnotationConfigApplicationContext(MainConfiguration.class);
```

## 2. 使用 @Bean 注册 Bean

### 2.1 基础注册方式
那么现在我们该如何配置Bean呢？

```java
@Configuration
public class MainConfiguration {

    @Bean("student")
    public Student student(){
        return new Student();
    }
}
```

通过 Java 代码为我们提供 Bean 配置，这样会更加的灵活，也更加便于控制 Bean 对象的创建。使用方法如下：

```java
ApplicationContext context = new AnnotationConfigApplicationContext(MainConfiguration.class);
Student student = context.getBean(Student.class);
System.out.println(student);
```

### 2.2 引入其他配置类 (@Import)
通过 `@Import` 还可以引入其他配置类（目前只做了解即可）：

```java
@Import(LBWConfiguration.class) 
@Configuration
public class MainConfiguration {
    // ...
}
```

## 3. Bean 属性与生命周期配置

### 3.1 常用属性配置
初始化方法和摧毁方法、自动装配可以直接在 `@Bean` 注解中进行配置。此外，还可以使用其他注解配置 lazy-init、scope 等属性：

```java
@Bean(name = "student", initMethod = "init", destroyMethod = "destroy", autowireCandidate = false)
@Lazy(true)     // 对应 lazy-init 属性
@Scope("prototype")    // 对应 scope 属性
@DependsOn("teacher")    // 对应 depends-on 属性
public Student student(){
    return new Student();
}
```

### 3.2 基于方法的依赖注入
对于那些我们需要通过构造方法(主类有参构造)或是 Setter(主类无参构造)完成依赖注入的 Bean，我们可以直接将其作为形式参数放到方法中：
```java
@Configuration
public class MainConfiguration {  //构造方法依赖注入
    @Bean
    public Teacher teacher(){
        return new Teacher();
    }

    @Bean
    public Student student(Teacher teacher) { //Setter依赖注入
        Student s = new Student();    // 1. 先调用无参构造创建对象
        s.setArtTeacher(teacher);     // 2. 再调用 setter 方法注入依赖
        return s;
    }
}
```
或者使用构造方法依赖注入

```java
@Configuration
public class MainConfiguration {  //构造方法依赖注入
    @Bean
    public Teacher teacher(){
        return new Teacher();
    }

    @Bean
    public Student student(Teacher teacher){
        return new Student(teacher);
    }
}
```

## 4. 自动装配详解

### 4.1 @Autowired 注解
除了基于构造器或是 Setter 的注入，我们也可以直接到 **Bean 对应的类(即Student.java)中使用自动装配**。`@Autowired` 可以用于字段、构造方法或 Setter 方法：

```java
public class Student {
    @Autowired   // 使用此注解来进行自动装配，由IoC容器自动为其赋值
    private Teacher teacher;
}
```

### 4.2 @Qualifier 解决歧义
`@Autowired` 默认采用 **byType** 的方式进行自动装配。如果出现了多个相同类型的 Bean，可以配合 `@Qualifier` 进行名称匹配：

```java
public class Student {
    @Autowired
    @Qualifier("a")   // 匹配名称为a的Teacher类型的Bean
    private Teacher teacher;
}
```

### 4.3 @Resource 注解
在 JDK 11 以后，`javax.annotation` 更名为 `jakarta.annotation`。`@Resource` 的作用与 `@Autowired` 相同，但机制有所不同：

- **@Resource**：默认 **byName**，如果找不到则 **byType**。可用于 set 方法、字段。
- **@Autowired**：默认 **byType**。可用于构造方法、set 方法、字段、方法参数。

使用 `@Resource` 需要导入依赖：

```xml
<dependency>
    <groupId>jakarta.annotation</groupId>
    <artifactId>jakarta.annotation-api</artifactId>
    <version>2.1.1</version>
</dependency>
```

### 4.4 生命周期注解
`@PostConstruct` 和 `@PreDestroy` 效果和 `init-method` 与 `destroy-method` 是一样的：

```java
@PostConstruct
public void init(){
    System.out.println("我是初始化方法");
}

@PreDestroy
public void destroy(){
    System.out.println("我是销毁方法");
}
```

## 5. 组件扫描与 @Component

### 5.1 @Component 注册方式
我们可以在类上添加 `@Component` 注解，并通过包扫描告诉 Spring 在哪些包中查找 Bean。这是目前最常用的方式：

```java
@Component("lbwnb")   // 同样可以自己起名字
public class Student {
}
```

在配置类中开启扫描：

```java
@Configuration
@ComponentScan("com.test.bean")   // 包扫描
public class MainConfiguration {
}
```

### 5.2 默认命名规则
无论是通过 `@Bean` 还是 `@Component` 注册，Spring 都会添加默认的 name 属性：
- **@Component**：类名首字母小写（如 Student 对应 student）。
- **@Bean**：对应的方法名称。

### 5.3 构造器自动注入
对于使用 `@Component` 注册的 Bean，如果其构造方法不是默认无参构造，那么默认会对其每一个参数都进行自动注入。

## 6. 进阶与总结

### 6.1 FactoryBean 接口
Spring 提供了 `FactoryBean` 接口，我们可以直接实现它来表示一个工厂 Bean：

```java
@Component
public class StudentFactory implements FactoryBean<Student> {
    @Override
    public Student getObject() {   // 生产的Bean对象
        return new Student();
    }

    @Override
    public Class<?> getObjectType() {   // 生产的Bean类型
        return Student.class;
    }

    @Override
    public boolean isSingleton() {   // 生产的Bean是否采用单例模式
        return false;
    }
}
```

### 6.2 XML 与注解的选择
官方说明指出，注解配置简洁且上下文信息丰富，而 XML 则在不修改源码的情况下配置组件方面更具优势。Spring 支持两种风格混合使用。

### 6.3 循环依赖思考
在 IoC 基础部分最后，留下一个关于循环依赖的问题：

```java
@Component
public class Student {
    @Resource
    private Teacher teacher;
}

@Component
public class Teacher {
    @Resource
    private Student student;
}
```
当两个类互相需要注入对方时，Spring 会如何处理？如果 Bean 变成原型模式，又会发生什么？这些问题将在实现原理部分详细介绍。
