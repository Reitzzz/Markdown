# 依赖注入 (Dependency Injection)



依赖注入（Dependency Injection，简称 DI）是一种设计模式，也是 Spring 框架的核心概念之一。前面我们了解了如何注册和使用一个 Bean，但为了彻底消除类之间的强关联（即实现解耦），我们还需要引入依赖注入。<br>
**依赖注入**就是指：一个对象所依赖的其他对象，不再由它自己去创建，而是由外部的容器在运行时自动“塞”给它。

## 1. 传统强耦合的痛点

假设我们有一个 `Teacher` 接口和两个具体实现类：

```java
public interface Teacher {
    void teach();
}

public class ArtTeacher implements Teacher {
    @Override
    public void teach() {
        System.out.println("我是美术老师，我教你画画！");
    }
}

public class ProgramTeacher implements Teacher {
    @Override
    public void teach() {
        System.out.println("我是编程老师，我教你学Golang！");
    }
}
```

在以前的开发模式中，如果 `Student` 需要一个老师，通常会直接 `new` 创建对应的对象：

```java
public class Student {
    // 在以前，如果我们需要指定哪个老师教我们，直接new创建对应的对象就可以了
    private Teacher teacher = new ArtTeacher();   
    
    public void study() {
        teacher.teach();
    }
}
```

**问题在于：** 如果现在不再让美术老师教，而是换成编程老师，我们就必须去修改 `Student` 类的源代码。当系统极其庞大且大量类都依赖 `Teacher` 时，这种硬编码的方式会导致牵一发而动全身，修改成本极高。

## 2. Setter 依赖注入 （无参构造）

有了依赖注入之后，`Student` 中的 `Teacher` 成员变量可以由 IoC 容器来选择一个合适的对象进行赋值。也就是说，IoC 容器在创建对象时，会将预先给定的属性注入到对象中。

要使用 Setter 依赖注入，类中对应的属性**必须有一个 set 方法**（无论成员变量的访问权限是什么），命名规则依然是驼峰命名法：

```java
public class Student {
    private Teacher teacher;
    
    // 要使用依赖注入，我们必须提供一个set方法
    public void setTeacher(Teacher teacher) {
        this.teacher = teacher;
    }
    
    public void study() {
        teacher.teach();
    }
}
```

在 XML 配置文件中，使用 `<property>` 标签来实现注入：

```xml
<bean name="teacher1" class="com.test.bean.ProgramTeacher"/>

<bean name="student" class="com.test.bean.Student">
    <property name="teacher" ref="teacher1"/>
</bean>
```

### 深入解析 `<property name="teacher" ref="teacher"/>`

这行代码的核心作用是告诉 Spring 容器：**使用 Setter 方法，将容器中另一个已经存在的 Bean 对象，注入到当前对象的属性中。** 我们可以将其拆解为三个核心部分来理解：

1. **`<property>` 标签：设定注入方式**
   这个标签明确告诉 Spring，你要使用 Setter 依赖注入。当 Spring 看到这个标签时，它就会知道必须通过调用目标类中的 `setXxx()` 方法来完成赋值，而不是通过构造方法。
2. **`name="teacher"`：指定目标属性**
   这里的 `name` 指的是当前 Java 类里面的属性名。Spring 底层会使用反射机制，将这个名字的首字母大写，并在前面拼上 `set`。也就是说，`name="teacher"` 会让 Spring 去当前的类中寻找 `setTeacher(...)` 这个方法。如果你的类里没有写这个 set 方法，程序启动时就会报错。
3. **`ref="teacher1"`：指定注入的对象源 (Reference)**
   这里的 `ref` 是 Reference（引用）的缩写，它指向的是 IoC 容器中另外一个 Bean 的名字。它告诉 Spring：“请去容器的大池子里找一个 name（或 id）叫做 `teacher1` 的 Bean，然后把它作为参数，传给刚刚找到的 `setTeacher()` 方法里。”

此时，Java 代码中不再出现任何具体的实现类信息。如果我们需要把老师换回 `ArtTeacher`，只需修改 XML 配置中 `teacher` 这个 Bean 的 `class` 即可，**无需改动任何 Java 源代码**。

除了注入对象（使用 `ref`），我们也可以直接注入一个**简单值**（即填写实体类本身的属性）：

```xml
<bean name="student" class="com.test.bean.Student">
    <property name="name" value="卢本伟"/>   //相当于student.setName("卢本伟");
</bean>
```

## 3. 构造方法依赖注入 （有参构造）

在很多情况下，类中的某些参数是在构造方法中就完成初始化的，比如 `final` 变量：

```java
public class Student {
    // 构造方法中完成初始化，所以是一个final变量
    private final Teacher teacher;   

    // Teacher属性是在构造方法中完成的初始化
    public Student(Teacher teacher) {   
        this.teacher = teacher;
    }
}
```

由于修改了默认的无参构造，IoC 容器默认只会调用无参构造，这会导致容器无法创建 Bean 并报错。我们需要使用 `<constructor-arg>` 标签指明使用的构造方法参数：

```xml
<bean name="teacher" class="com.test.bean.ArtTeacher"/>

<bean name="student" class="com.test.bean.Student">
    <constructor-arg name="teacher" ref="teacher"/>
</bean>
```

如果类中有多个构造方法造成歧义，可以通过 `type` 或 `name` 来精确匹配目标构造方法：

```java
public class Student {
    private final String name;
    
    public Student(String name) {
        System.out.println("我是一号构造方法");
        this.name = name;
    }

    public Student(int age) {
        System.out.println("我是二号构造方法");
        this.name = String.valueOf(age);
    }
}
```

通过 `type` 指定参数类型：
```xml
<constructor-arg value="1" type="int"/>    //相当于Student student = new Student(1);
```

通过 `name` 指定参数名称：
```xml
<constructor-arg value="1" name="age"/>   
```

## 4. 集合类型的注入

Spring 对于集合类型（如 List、Set、Map、数组等）有着特殊的配置支持。

### 注入 List 集合

```java
public class Student {
    private List<String> list;

    public void setList(List<String> list) {
        this.list = list;
    }
}
```

```xml
<bean name="student" class="com.test.bean.Student">
    <property name="list">
        <list>
            <value>AAA</value>
            <value>BBB</value>
            <value>CCC</value>
        </list>
    </property>
</bean>
```

### 注入 Map 集合

Map 类型可以使用 `<map>` 和 `<entry>` 标签进行注入：

```xml
<bean name="student" class="com.test.bean.Student">
    <property name="map">
        <map>
            <entry key="语文" value="100.0"/>
            <entry key="数学" value="80.0"/>
            <entry key="英语" value="92.5"/>
        </map>
    </property>
</bean>
```

## 总结

至此，我们完成了两种核心依赖注入方式的学习：
1. **Setter 依赖注入：** 通过成员属性对应的 `set` 方法完成注入（使用 `<property>`）。
2. **构造方法依赖注入：** 通过构造方法完成注入（使用 `<constructor-arg>`）。
