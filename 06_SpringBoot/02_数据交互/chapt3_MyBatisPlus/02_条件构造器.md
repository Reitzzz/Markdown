MyBatis-Plus 提供了一套强大的条件构造器（Wrapper），用于构建复杂的数据库查询条件。
官方网站地址：https://baomidou.com/guides/wrapper/

## 2.1 QueryWrapper复杂查询

对于一些复杂查询的情况，MybatisPlus支持我们自己构造QueryWrapper用于复杂条件查询：

```java
@Test
void contextLoads() {
    QueryWrapper<User> wrapper = new QueryWrapper<>();    // 复杂查询可以使用QueryWrapper来完成
    wrapper
            .select("id", "name", "email", "password")    // 可以自定义选择哪些字段 不填写则相当于 *
            .ge("id", 2)                // 选择判断id大于等于1的所有数据
            .orderByDesc("id");   // 根据id字段进行降序排序
    System.out.println(mapper.selectList(wrapper));   // Mapper同样支持使用QueryWrapper进行查询
}
```

通过使用上面的QueryWrapper对象进行查询，也就等价于下面的SQL语句：

```sql
select id,name,email,password from user where id >= 2 order by id desc
```

我们可以在配置中开启SQL日志打印：

```yaml
mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

## 2.2 批处理操作

有些时候我们遇到需要批处理的情况，也可以直接使用批处理操作：

```java
@Test
void contextLoads() {
    // 支持批处理操作，我们可以一次性删除多个指定ID的用户
    int count = mapper.deleteBatchIds(List.of(1, 3));   // 参数为集合 表示删除id为1和3的用户
    System.out.println(count);
}
```

## 2.3 分页查询

分页功能在实际应用中非常重要。

### 2.3.1 分页的优势

**1. 解决大数据量问题**
假设数据库中有 100万条用户记录：
* **不分页**：一次性加载 100万条数据。占用大量内存，网络传输巨大，服务器响应极慢，极易导致内存溢出 (OOM)。
* **分页**：只加载当前页（如10条）。内存占用极小，网络传输快，服务器响应迅速。

**2. 改善用户体验**
通过将数据拆分为第1页、第2页等，用户可以：
* 快速浏览核心内容。
* 减少等待页面加载的时间。
* 方便地通过翻页导航定位数据。

### 2.3.2 实际应用场景

| 场景 | 说明 |
| --- | --- |
| 电商产品列表 | 商品太多，分页显示，每页10或20条 |
| 评论列表 | 微博/知乎等评论分页加载 |
| 后台管理系统 | 用户、订单等列表数据分页展示 |
| 搜索引擎 | Google 搜索结果分页（第1页10条，第2页10条...） |
| 日志查询 | 系统日志可能有百万级，分页便于查询 |

### 2.3.3 分页配置与基本使用

在执行分页前，需要先配置分页拦截器：

```java
@Configuration
public class MybatisConfiguration {
    @Bean
    public MybatisPlusInterceptor paginationInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        // 添加分页拦截器到MybatisPlusInterceptor中
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
        return interceptor;
    }
}
```

测试类基本用法：

```java
@Test
void contextLoads() {
    // Page.of(current, size): current为当前页码（从1开始），size为每页记录数
    Page<User> page = mapper.selectPage(Page.of(1, 2), Wrappers.emptyWrapper());  
    
    System.out.println(page.getRecords());   // 获取分页之后的数据内容
    System.out.println(page.getTotal());     // 获取数据总条数
}
```

### 2.3.4 项目实战：三层架构封装分页

在实际后端开发中，分页参数通常由前端传来，我们需要在 Service 层完成逻辑封装。

**Service 接口层：**
```java
public interface UserService extends IService<User> {
    // 根据关键字分页查询用户
    Page<User> getUserPage(Integer current, Integer size, String keyword);
}
```

**Service 实现层：**
```java
@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService {
    @Override
    public Page<User> getUserPage(Integer current, Integer size, String keyword) {
        // 1. 创建分页对象：current页，展示size条
        Page<User> page = Page.of(current, size); 
        
        // 2. 构造查询条件
        LambdaQueryWrapper<User> wrapper = Wrappers.lambdaQuery();
        if (keyword != null && !keyword.isEmpty()) {
            wrapper.like(User::getName, keyword); // 模糊查询
        }
        wrapper.orderByDesc(User::getId);  // 按照ID降序排列
        
        // 3. 执行分页查询（ServiceImpl内置了page方法）
        return this.page(page, wrapper);
    }
}
```

**Controller 层：**
```java
@RestController
@RequestMapping("/user")
public class UserController {
    @Autowired
    private UserService userService;

    @GetMapping("/page")
    public Result<Page<User>> pageQuery(
            @RequestParam(defaultValue = "1") Integer current,
            @RequestParam(defaultValue = "10") Integer size,
            @RequestParam(required = false) String keyword) {
        Page<User> pageInfo = userService.getUserPage(current, size, keyword);
        return Result.success(pageInfo); // 返回包含 records(数据) 和 total(总条数) 的对象
    }
}
```

## 2.4 UpdateWrapper数据更新

对于数据更新操作，我们也可以使用UpdateWrapper非常方便地来完成：

```java
@Test
void contextLoads() {
    UpdateWrapper<User> wrapper = new UpdateWrapper<>();
    wrapper
            .set("name", "lbw")
            .eq("id", 1);
    System.out.println(mapper.update(null, wrapper));
}
```

### 常见的条件方法

| 方法 | SQL对应 | 说明 |
| --- | --- | --- |
| `.eq("id", 1)` | `WHERE id = 1` | 等于 |
| `.ne("id", 1)` | `WHERE id != 1` | 不等于 |
| `.gt("id", 5)` | `WHERE id > 5` | 大于 |
| `.ge("id", 5)` | `WHERE id >= 5` | 大于等于 |
| `.like("name", "张")` | `WHERE name LIKE '%张%'` | 模糊查询 |
| `.in("id", 1,2,3)` | `WHERE id IN (1,2,3)` | 包含 |

## 2.5 Lambda条件构造器

专门支持 Java 8 Lambda 表达式的实现，避免硬编码字段名，提高维护性：

```java
@Test
void contextLoads() {
        LambdaQueryWrapper<User> wrapper = Wrappers
                .<User>lambdaQuery()
                .eq(User::getId, 2)   // 传入方法引用，自动解析字段名
                .select(User::getName, User::getId); 
        System.out.println(mapper.selectOne(wrapper));
}
```
