# Git 基础操作与 Git Graph 提效指南

## 1. Git 核心命令行操作

理解命令行有助于掌握 Git 的核心工作流（工作区 -> 暂存区 -> 本地仓库 -> 远程仓库）。

### 初始化与获取
* git init：在当前目录创建一个新的本地仓库
* git clone <url>：从远程服务器完整下载一个仓库到本地

### 提交与同步
* git status：查看当前文件的修改和暂存状态
* git add .：将当前目录下的所有修改添加到暂存区
* git commit -m "提交信息"：将暂存区的内容正式提交到本地仓库
* git pull origin <分支名>：从远程仓库拉取最新代码并与本地合并
* git push origin <分支名>：将本地仓库的更新推送到远程仓库

### 分支管理
* git branch：列出本地所有的分支
* git branch <分支名>：创建一个新分支
* git checkout <分支名>：切换到指定分支
* git merge <分支名>：将指定分支的代码合并到当前所在分支

---

## 2. Git Graph (VS Code) 可视化操作

Git Graph 可以将枯燥的命令行转化为直观的图形化点击操作。在 VS Code 中，可通过点击底部状态栏或“源代码管理”视图顶部的图标打开。

### 查看历史与代码对比 (对应 git log 与 git diff)
* 操作：点击图表中代表 Commit 的圆点节点，底部或侧边面板会显示修改的文件列表。直接点击文件名，即可在编辑器中查看清晰的代码增删对比。

### 切换与新建分支 (对应 git checkout 与 git branch)
* 切换分支：在图中找到目标分支的标签，双击该标签，或右键点击标签并选择 Checkout Branch。
* 新建分支：右键点击希望作为起点的历史节点，选择 Create Branch。在输入框中命名，勾选相关选项可实现在创建后直接切换过去。

### 合并代码 (对应 git merge)
* 操作：首先确保当前处于接收代码的分支上（例如主分支）。找到想要合并进来的分支，右键点击该分支的最新节点，选择 Merge into current branch。

### 撤销与回退 (对应 git reset)
* 操作：如果需要回退提交，右键点击想回退到的历史安全节点，选择 Reset current branch to this Commit。界面会提供 Soft、Mixed 或 Hard 模式供选择，降低误操作风险。

### 暂存与提交 (对应 git add 与 git commit)
* 操作：日常的暂存和提交通常直接使用 VS Code 左侧自带的“源代码管理”面板。点击文件旁的 + 号进行暂存，在上方文本框输入信息并点击提交。提交后，Git Graph 的图形树会自动刷新并长出一个新节点。
