# Git Graph (VS Code) 核心与进阶操作指南

Git Graph 是 Visual Studio Code 中极具效率的插件，它将枯燥的命令行变成了直观的点击操作。安装后，你可以通过点击 VS Code 底部状态栏的 Git Graph 按钮，或者在左侧“源代码管理”视图顶部点击对应的图形图标来打开它。

以下是如何用 Git Graph 结合 VS Code 原生功能进行操作，涵盖了日常开发与团队协作中的进阶场景：

## 1. 基础工作流

* **查看历史与代码对比**
  * 操作：界面中会直接展示由节点和连线构成的分支树。点击任意一个圆点（Commit 节点），底部或侧边会滑出面板，显示该次提交修改了哪些文件。直接点击文件名，即可查看清晰的代码差异对比。
* **切换分支**
  * 操作：在图中找到目标分支的标签（通常带有颜色），双击该标签，或者右键点击它并选择 Checkout Branch。
* **新建分支**
  * 操作：右键点击你希望作为基础的那个历史节点，选择 Create Branch。在弹出的输入框中命名（例如 feature-user-login），勾选对应的选项可以直接在创建后切换过去。
* **暂存与提交代码**
  * Git Graph 主要用于查看历史和分支管理，具体的暂存与提交动作最标准的做法是结合 VS Code 左侧自带的“源代码管理”面板：
  * **第一步：暂存** * 你的所有代码修改都会出现在“更改”列表中。将鼠标悬停在你想暂存的文件（或整个文件夹）上，点击右侧出现的 + 号（暂存更改）。此时，这些文件会移动到“暂存的更改”列表中。这相当于告诉 Git：“我已经确认这些文件的修改，准备打包了”。
  * **第二步：提交**
    * 确认文件都已暂存后，在面板上方的文本框中输入本次修改的总结说明（Commit Message），然后点击下方的“提交”按钮。提交完成后，回到 Git Graph 视图，你会看到分支树上立刻长出了一个属于你的新节点。

## 2. 代码同步与合并

* **获取远程更新**
  * 操作：点击 Git Graph 顶部工具栏的云端刷新图标（Fetch from Remote）。这会把远程仓库最新的提交节点拉取到图表中显示，但不会修改你本地的工作区代码，方便你先观察团队其他人的进度。
* **拉取并合并**
  * 操作：右键点击远程分支的最新节点（如 origin/main），选择 Pull into current branch。
* **合并代码**
  * 操作：首先确保你当前处于接收代码的分支上（例如 main）。然后找到你想合并进来的分支（例如你刚开发完的业务模块分支），右键点击该分支的最新节点，选择 Merge into current branch。
* **推送到远程**
  * 操作：本地提交生成新节点后，右键点击该节点所在的分支标签，选择 Push Branch。如果是新建的本地分支，系统会提示你在远程建立对应的追踪分支。

## 3. 进阶与挽救操作

* **撤销与回退**
  * 操作：如果某次提交出了问题，右键点击你想回退到的那个历史安全节点，选择 Reset current branch to this Commit。
  * **三种模式与后续操作**：
    * **Soft (软回退)**：撤销的修改会直接退回到“暂存区”。后续无需重新暂存，直接在源代码管理面板填写新的提交信息并点击“提交”即可。
    * **Mixed (混合回退 - 默认)**：撤销的修改会退回到“工作区”（未暂存状态）。后续需要先点击文件旁的 + 号重新暂存，再填写信息并提交。
    * **Hard (硬回退)**：撤销的代码会被彻底清空，文件恢复到历史安全节点的最初模样。后续无需提交，直接在这个干净的旧版本上重新编写代码即可。
  * **强制推送提醒**：如果回退的提交之前已经 push 到了远程仓库，本地回退并重新提交后，需要使用强制推送来覆盖远程记录。在源代码管理面板，点击上方 ... (更多操作) -> 拉取、推送 -> 强制推送 (Push (Force))。
* **储藏临时进度**
  * 场景：比如你正在编写某个复杂逻辑，代码写了一半还没法提交，此时突然需要切回主分支紧急修复一个代码 Bug。
  * 操作：在左侧“源代码管理”面板的更多选项（...）中选择 储藏 -> 储藏 (Stash)。这会把你当前没写完的代码临时存起来。修完 Bug 切回原分支后，再选择 储藏 -> 弹出储藏 (Pop Stash) 恢复工作现场。
* **拣选特定提交**
  * 场景：你只需要另一个分支上的某一个特定修复或小功能，而不需要合并整个分支。
  * 操作：在图表中找到那个特定的 Commit 节点，右键点击它，选择 Cherry Pick。这个操作会把该节点的修改单独“复制”并应用到你当前的分支上。
* **变基整理历史**
  * 场景：为了保持提交历史的绝对线性，在多人协作合并代码前，有时会要求使用 Rebase 而不是直接 Merge。
  * 操作：在你想变基到的目标节点上右键，选择 Rebase current branch on this Commit。如果在变基过程中出现代码冲突，VS Code 会在编辑器中高亮提示，解决冲突并暂存文件后，在源代码管理面板点击继续变基即可。
