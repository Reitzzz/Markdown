## 概念
- 线程与进程：进程是程序的基本执行实体；线程是操作系统能够进行运算调度的最小单位，被包含在进程之中，是进程中的实际执行单位。
- 并发与并行：并发指在同一时刻，有多个指令在单个CPU上交替执行；并行指在同一时刻，有多个指令在多个CPU上同时执行。

## 实现方式

### 1. 继承Thread类
方法：定义一个类继承Thread，重写run()方法，创建子类对象，并启动线程。
优缺点：编程简单；可扩展性较差 不能继承其他类。
特点：在卖票问题中需要使用static修饰票数。
方法声明：public void start()
```java
class MyThread extends Thread {
    @Override
    public void run() {
        
    }
}

MyThread t1 = new MyThread();
t1.start();
```

### 2. 实现Runnable接口
方法：定义一个类实现Runnable，重写run()方法，创建自己的类的对象，创建一个Thread类的对象并开启线程。
优缺点：扩展性强 可以继承其他类；编程相对复杂。
特点：在卖票问题中不需要使用static修饰票数。
方法声明：public Thread(Runnable target)
```java
class MyRun implements Runnable {
    @Override
    public void run() {
        
    }
}

MyRun mr = new MyRun();
Thread t1 = new Thread(mr);
t1.start();
```

### 3. 利用Callable接口和Future接口
方法：创建一个类实现Callable接口(存在泛型 即结果类型)，重写call方法(有返回值 表示多线程运行的结果)，创建自己的类的对象(表示多线程要执行的任务)，创建FutureTask的对象(作用管理多线程运行的结果)，创建Thread类的对象，启动。
优缺点：扩展性强 可以继承其他类；编程相对复杂。
特点：可以获取到多线程运行的结果。
方法声明：public FutureTask(Callable<V> callable)
```java
class MyCall implements Callable<String> {
    @Override
    public String call() throws Exception {
        return "Result";
    }
}

MyCall mc = new MyCall();
FutureTask<String> ft = new FutureTask<>(mc);
Thread t1 = new Thread(ft);
t1.start();
```

## 常见成员方法

- 获取名称。如果没有给线程设置名字，线程也有默认名字Thread-X(X序号，从0开始)。
方法声明：public final String getName()

- 设置名称
方法声明：public final void setName(String name)

- 获取当前线程
方法声明：public static Thread currentThread()

- 线程休眠。单位为毫秒ms。
方法声明：public static void sleep(long time)

- 设置优先级。最小为1，最大为10，默认为5。
方法声明：public final void setPriority(int newPriority)

- 获取优先级
方法声明：public final int getPriority()

- 设置守护线程。依赖性运行：守护线程的运行完全依附于系统中的用户线程。只要程序中还有一个非守护线程（用户线程）在运行，守护线程就会持续工作。强制自动退出：当所有的用户线程都执行完毕并结束时，程序进程会自动关闭。此时，无论守护线程是否正在执行任务，它们都会立刻被系统强制终止。程序不会等待守护线程完成手头的工作。
方法声明：public final void setDaemon(boolean on)

- 出让执行权
方法声明：public static void yield()

- 加入线程。放在start()之后调用。等到join()的线程全部运行完毕之后 其他线程才执行。
方法声明：public final void join()

## 线程安全问题
线程执行代码的时候，CPU的执行权随时有可能会被其他的线程给抢走。

代码步骤：1.循环 2.同步代码块 3.判断共享数据是否到了末尾（到了末尾） 4.判断共享数据是否到了末尾（没有到末尾执行核心逻辑)

### 同步代码块
格式：synchronized(锁对象){ 操作共享数据的代码块 }。锁对象可以任意，但必须使用static修饰，如static Object obj=new Object(); 一般使用MyThread.class。
特点：1. 锁默认打开，有一个线程进去了，锁自动关闭；2. 里面的代码全部执行完毕，线程出来，锁自动打开。
细节：1.必须写在循环内部；2.锁对象必须唯一；3.如果使用while(true) 则需要在锁的最后加上睡眠 强制让当前进程交出CPU。
```java
static Object obj = new Object();

synchronized(obj) {
    
}
```

### 同步方法
使用场景：需要将全部代码都锁起来。
特点：1.锁住方法内部所有代码；2.锁对象不能指定。非静态: this；静态: 当前类的字节码文件对象。
```java
public synchronized void method() {
    
}
```

本质是接口，需要通过其实现类ReentrantLock来实例化。
方法声明：public void lock()
方法声明：public void unlock()
```java

lock.lock();
try {

    lock.unlock();

```

## 生产者与消费者
即等待-唤醒机制。

### 方法1
- wait()
- notify() (随机唤醒绑定在锁上的某个进程)

### 方法2：阻塞队列
概念：阻塞 (put失败-wait 即阻塞; take失败-wait 即阻塞)。
继承结构：接口 (Iterable, Collection, Queue, BlockingQueue)。
实现类：
- ArrayBlockingQueue：底层是数组 有界。其中推荐写<String> 但不写也可编译。
```java
ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(1);
```
- LinkedBlockingQueue：底层是链表 不需要指定界限 界限为int最大值。

## 线程池

### 工具类实现
1. 创建线程池
- Executors.newCachedThreadPool()。虽然说是“没有上限”，但它的最大线程数其实是 Integer.MAX_VALUE。如果任务执行很慢且任务量巨大，可能会瞬间开启大量线程导致 OOM (Out Of Memory)。
- Executors.newFixedThreadPool(3)。固定线程数，适合已知 CPU 核心数或资源有限的场景。它的任务队列是 LinkedBlockingQueue，也是要注意队列积压导致的内存问题。

2. 提交任务
- pool.submit(Runnable task)：提交任务并返回一个 Future 对象。
- pool.execute(Runnable command)：提交不需要返回值的任务。

3. 任务执行完毕 关闭线程池
- pool.shutdown()：平滑关闭线程池。调用后不再接受新任务，但会把等待队列里已经提交的任务执行完毕。
- pool.shutdownNow()：强制关闭线程池。尝试中断正在执行的任务，并返回队列中尚未执行的任务列表。

### 自定义实现
参数一：核心线程数量 不能小于0
参数二：最大线程数 不能小于0，最大数量 >= 核心线程数量
参数三：空闲线程最大存活时间 不能小于0
参数四：时间单位 用TimeUnit指定 (TimeUnit.SECONDS)
参数五：任务队列 不能为null (即阻塞队列: ArrayBlockingQueue, LinkedBlockingQueue)
参数六：创建线程工厂 不能为null (即线程池如何获取到一个线程: Executors.defaultThreadFactory())
参数七：任务的拒绝策略 不能为null (new ThreadPoolExecutor.AbortPolicy())

细节：
1. 只有当核心线程忙碌 队伍排满时 才会创建临时线程
2. 不一定按照提交顺序执行

```java
ThreadPoolExecutor threadPoolExecutor = new ThreadPoolExecutor(
    3,
    6,
    60,
    TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(3),
    Executors.defaultThreadFactory(),
    new ThreadPoolExecutor.AbortPolicy()
);
```

### 如何确定线程池大小
获取最大并行数：
```java
System.out.println(Runtime.getRuntime().availableProcessors());
```

运算：
- CPU密集型运算：\text{最大并行数} + 1
- I/O密集型运算：\text{最大并行数} \times \text{期望CPU利用率} \times \frac{\text{计算时间}+\text{等待时间}}{\text{计算时间}}
