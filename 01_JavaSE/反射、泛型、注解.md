## 反射

### 概念
反射允许对成员变量、成员方法和构造方法的信息进行编程访问。

### 1. 获取Class对象
- 源代码阶段
方法声明：public static Class<?> forName(String className)
```java
Class<?> clazz1 = Class.forName("com.example.Student");
```

- 加载阶段
```java
Class<?> clazz2 = Student.class;
```

- 运行阶段
方法声明：public final Class<?> getClass()
```java
Student stu = new Student();
Class<?> clazz3 = stu.getClass();
```

### 2. 公共解剖方法

- 获取修饰符
方法声明：public int getModifiers()
```java
int modifiers = clazz1.getModifiers();
```

- 获取名字
方法声明：public String getName()
```java
String name = clazz1.getName();
```

- 获取形参/类型
构造方法与成员方法获取形参对象数组：
方法声明：public Parameter[] getParameters()
字段获取数据类型对象：
方法声明：public Class<?> getType()

### 3. 各类成员的专属获取与运行方法

#### 构造方法
- 获取所有公共构造方法 / 所有构造方法
方法声明：public Constructor<?>[] getConstructors()
方法声明：public Constructor<?>[] getDeclaredConstructors()
```java
Constructor<?>[] constructors = clazz1.getConstructors();
```

- 获取单个公共构造方法 / 单个构造方法
方法声明：public Constructor<T> getConstructor(Class<?>... parameterTypes)
方法声明：public Constructor<T> getDeclaredConstructor(Class<?>... parameterTypes)
```java
Constructor<?> constructor = clazz1.getDeclaredConstructor(String.class, int.class);
```

- 取消访问检查与创建对象
方法声明：public void setAccessible(boolean flag)
方法声明：public T newInstance(Object... initargs)
```java
constructor.setAccessible(true);
Object obj = constructor.newInstance("张三", 20);
```

#### 成员变量
- 获取所有公共成员变量 / 所有成员变量
方法声明：public Field[] getFields()
方法声明：public Field[] getDeclaredFields()
```java
Field[] fields = clazz1.getDeclaredFields();
```

- 获取单个公共成员变量 / 单个成员变量
方法声明：public Field getField(String name)
方法声明：public Field getDeclaredField(String name)
```java
Field field = clazz1.getDeclaredField("age");
```

- 修改变量与获取值
方法声明：public void set(Object obj, Object value)
方法声明：public Object get(Object obj)
```java
field.setAccessible(true);
field.set(obj, 25);
Object value = field.get(obj);
```

#### 成员方法
- 获取所有公共成员方法(包括继承的) / 所有成员方法(不包括继承的)
方法声明：public Method[] getMethods()
方法声明：public Method[] getDeclaredMethods()
```java
Method[] methods = clazz1.getDeclaredMethods();
```

- 获取单个公共成员方法 / 单个成员方法
方法声明：public Method getMethod(String name, Class<?>... parameterTypes)
方法声明：public Method getDeclaredMethod(String name, Class<?>... parameterTypes)
```java
Method method = clazz1.getDeclaredMethod("study", String.class);
```

- 运行方法
方法声明：public Object invoke(Object obj, Object... args)
```java
method.setAccessible(true);
Object result = method.invoke(obj, "Java");
```

## 泛型

### 引入
有苹果，香蕉，梨。分别定义对应的箱子类太多太繁杂；使用Object箱子，取出水果后做强制转换又不能保证结果正确。

### 使用

- 泛型类
```java
public class GenericBox<T> {
    private T content;
    public T getContent() { return content; }
    public void setContent(T content) { this.content = content; }
}

GenericBox<Apple> genericBox = new GenericBox<>();
genericBox.setContent(new Apple());
Apple content = genericBox.getContent();
```

- 泛型接口
```java
public interface Generator<T> {
    T next();
}

public class RandomGenerator implements Generator<String> {
    public String next() {
        return "Random String";
    }
}
```

- 泛型方法
```java
public class Utility {
    public static <E> void printArray(E[] array) {
        for (E element : array) {
            System.out.println(element);
        }
    }
}
```

### 通配符

通配符就是为了解决泛型不可变性的“死板”问题而诞生的。它们在保证类型安全的前提下，提升了API的灵活性。

- 上界通配符 (`? extends T`)
意义: 规定了泛型类型的上限。表示这是一个具体的类型，但这个类型是T或者T的子类。允许读取，禁止写入。
```java
List<Apple> apples = new ArrayList<>();
List<? extends Fruit> fruits = apples;
Fruit f = fruits.get(0);
```

- 下界通配符 (`? super T`)
意义: 规定了泛型类型的下限。表示这是一个具体的类型，但这个类型是T或者T的父类。读取受限，允许写入。
```java
List<Food> foods = new ArrayList<>();
List<? super Apple> applesList = foods;
applesList.add(new Apple());
Object obj = applesList.get(0);
```

- 无界通配符 (`?`)
代表未知类型。只关心这个集合作为一个容器本身的属性（比如大小、是否为空），而不关心容器里面装的到底是什么。
```java
public void printListSize(List<?> list) {
    System.out.println(list.size());
    list.clear();
}
```

## 注解

### 作用
不是程序本身，但可以对程序作出解释，且可以被其他程序读取。

### 使用
格式：`@注释名`，可以添加一些参数值。如 `@SuppressWarnings(value="unchecked")`。
位置：可以附加在package，class，method，field的上面，相当于给他们添加了额外的辅助信息。

### 种类

#### 内置注解
- `@Override`：重写。
- `@Deprecated`：表示不鼓励使用。
- `@SuppressWarnings`：需要添加参数才能正确使用。`"all"` 忽略所有类型警告；`"unchecked"` 忽略执行未检查的转换时的警告。

#### 元注解
- `@Target()`：用于描述注解的使用范围。参数有 `ElementType.TYPE` (类、接口或枚举), `METHOD`, `PARAMETER`, `FIELD`。
- `@Retention`：表示注解在代码编译、加载和运行的哪个阶段依然有效。SOURCE < CLASS < RUNTIME (最常用，JVM加载后依然存在，可通过反射获取)。
- `@Documented`：说明该注解将被包含在javadoc中。
- `@Inherited`：说明子类可以继承父类中的该注解。

#### 自定义注解
参数类型+方法名+default。

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@interface MyAnnotation2 {
    String name() default "";
    int age() default 0;
    int id() default -1;
    int[] arr() default {};
}

public class Demo3 {
    @MyAnnotation2(name = "lc")
    public void method() {
    }
}
```
