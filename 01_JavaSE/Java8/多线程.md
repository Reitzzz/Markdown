## 1. 概念
- **线程与进程**：进程是程序的基本执行实体；线程是操作系统调度的最小单位，包含在进程中。
- **并发与并行**：并发指多个指令在单 CPU 上交替执行；并行指多个指令在多 CPU 上同时执行。

## 2. 实现方式

### 2.1 继承 Thread 类
定义类继承 `Thread`，重写 `run()`，创建对象并调用 `start()`。  
优点：编程简单；缺点：扩展性差。  
卖票问题中票数需用 `static` 修饰。

```java
class MyThread extends Thread {
    @Override
    public void run() {
        // 线程执行体
    }
}
MyThread t1 = new MyThread();
t1.start();
```

### 2.2 实现 Runnable 接口
定义类实现 `Runnable`，重写 `run()`，创建 `Thread` 对象并启动。  
优点：可继承其他类；缺点：相对复杂。  
卖票问题中票数无需 `static` 修饰。

```java
class MyRun implements Runnable {
    @Override
    public void run() {
        // 线程执行体
    }
}
MyRun mr = new MyRun();
Thread t1 = new Thread(mr);
t1.start();
```

### 2.3 利用 Callable 和 Future 接口
实现 `Callable` 接口（泛型为返回结果类型），重写 `call()` 方法。  
优点：可获取返回值、抛出异常；缺点：相对复杂。

```java
class MyCall implements Callable<String> {
    @Override
    public String call() throws Exception {
        return "Result";
    }
}
MyCall mc = new MyCall();
FutureTask<String> ft = new FutureTask<>(mc);
Thread t1 = new Thread(ft);
t1.start();
String result = ft.get(); // 获取返回值
```

## 3. 常见成员方法

| 方法 | 说明 |
|------|------|
| `getName()` / `setName(String)` | 获取/设置线程名称（默认 `Thread-X`） |
| `currentThread()` | 静态方法，获取当前线程对象 |
| `sleep(long ms)` | 静态方法，休眠指定毫秒 |
| `setPriority(int)` / `getPriority()` | 设置/获取优先级（1~10，默认 5） |
| `setDaemon(boolean)` | 设置为守护线程（用户线程结束后自动退出） |
| `yield()` | 静态方法，出让 CPU 执行权 |
| `join()` | 等待该线程执行完毕 |

## 4. 线程安全问题

### 4.1 同步代码块
```java
synchronized(锁对象) {
    // 操作共享数据的代码
}
```
- 锁对象必须唯一（可用 `类名.class`）。
- 必须写在循环内部；若 `while(true)` 需在锁内加 `sleep` 释放 CPU。

### 4.2 同步方法
```java
public synchronized void method() { }
```
- 非静态方法锁对象为 `this`，静态方法锁对象为 `类名.class`。

### 4.3 Lock锁
```java
Lock lock = new ReentrantLock();
lock.lock();
try {
    // 操作共享数据
} finally {
    lock.unlock();
}
```

### 4.4 死锁
两个或以上线程互相持有对方需要的锁，导致永久阻塞。避免嵌套锁。

## 5. 生产者与消费者

### 5.1 wait/notify 机制
- `wait()`：释放锁并等待。
- `notify()`：随机唤醒一个等待线程。
- `notifyAll()`：唤醒所有等待线程。

### 5.2 阻塞队列
- **ArrayBlockingQueue**：有界，底层数组。
- **LinkedBlockingQueue**：无界（`Integer.MAX_VALUE`），底层链表。

```java
BlockingQueue<String> queue = new ArrayBlockingQueue<>(10);
queue.put("data");   // 满则阻塞
String data = queue.take(); // 空则阻塞
```

## 6. 线程池

### 6.1 工具类实现
- `Executors.newCachedThreadPool()`：无界线程池（注意 OOM 风险）。
- `Executors.newFixedThreadPool(int)`：固定大小线程池。
- `submit()` 提交任务返回 `Future`，`execute()` 提交无返回值任务。
- `shutdown()` 平滑关闭，`shutdownNow()` 强制关闭。

### 6.2 自定义实现
```java
ThreadPoolExecutor pool = new ThreadPoolExecutor(
    3,                           // 核心线程数
    6,                           // 最大线程数
    60, TimeUnit.SECONDS,        // 空闲线程存活时间
    new ArrayBlockingQueue<>(3), // 任务队列
    Executors.defaultThreadFactory(),
    new ThreadPoolExecutor.AbortPolicy() // 拒绝策略
);
```
- 只有核心线程忙碌且队列满时，才会创建临时线程。

### 6.3 线程池大小确定
- CPU 密集型：`CPU 核心数 + 1`
- I/O 密集型：`核心数 × 期望利用率 × (计算时间 + 等待时间) / 计算时间`

获取核心数：`Runtime.getRuntime().availableProcessors()`

## 8. 线程状态与 Synchronized 锁升级

### 8.1 Java 线程的 6 种状态
| 状态 | 说明 |
|------|------|
| NEW | 创建未启动 |
| RUNNABLE | 运行中（含 Ready 和 Running） |
| BLOCKED | 等待获取锁 |
| WAITING | 无限期等待（`wait/join/park`） |
| TIMED_WAITING | 限时等待（`sleep/wait(ms)`） |
| TERMINATED | 执行结束 |

### 8.2 Synchronized 锁升级过程
偏向锁 → 轻量级锁 → 重量级锁

| 阶段 | 适用场景 | 实现原理 |
|------|----------|----------|
| 偏向锁 | 单线程反复获取锁 | Mark Word 记录线程 ID |
| 轻量级锁 | 多线程交替执行，无竞争 | CAS 修改 Mark Word，自旋等待 |
| 重量级锁 | 多线程并发竞争激烈 | 操作系统互斥量，线程挂起 |

### 8.3 wait() 与 sleep() 对比
| 方法 | 释放锁 | 所属类 | 唤醒条件 |
|------|--------|--------|----------|
| `wait()` | 释放 | Object | `notify()/notifyAll()` |
| `sleep()` | 不释放 | Thread | 时间到 / `interrupt()` |

### 8.4 线程中断机制
- `interrupt()`：设置中断标志。
- `isInterrupted()`：检查标志（不清除）。
- `interrupted()`：静态方法，检查并清除标志。
- 阻塞状态下收到中断会抛出 `InterruptedException` 并清除标志。

### 8.5 死锁排查命令
```bash
jps -l          # 查看 Java 进程 PID
jstack <pid>    # 打印线程堆栈，末尾显示死锁信息
jconsole        # 图形化监控工具
```

## 9. ThreadLocal
为每个线程提供独立变量副本，互不干扰。常用于存储线程上下文（如数据库连接、用户 Session）。  
使用后务必调用 `remove()` 防止内存泄漏。

```java
ThreadLocal<String> local = new ThreadLocal<>();
local.set("value");
String val = local.get();
local.remove();
```

**InheritableThreadLocal**：子线程可继承父线程的值。

## 10. Timer 定时器
```java
Timer timer = new Timer();
timer.schedule(new TimerTask() {
    @Override
    public void run() {
        System.out.println("任务执行");
    }
}, 1000, 2000); // 延迟1秒，每2秒执行
timer.cancel(); // 取消
```
推荐使用 `ScheduledExecutorService` 替代。

## 11. 虚拟线程（Java 21+）
轻量级线程，由 JVM 调度，适用于高并发 I/O 密集型任务。默认守护线程。

```java
// 方式一
Thread.startVirtualThread(() -> { });

// 方式二
Thread.ofVirtual().name("v-thread").start(() -> { });

// 方式三（线程池）
try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> { });
}
```
> 注意：避免在虚拟线程中使用 `synchronized` 长时间阻塞，推荐用 `ReentrantLock`。

## 12. 线程生成器（Java 21+）
快速创建并配置平台线程。

```java
Thread.ofPlatform()
      .name("worker")
      .daemon(true)
      .priority(Thread.MAX_PRIORITY)
      .uncaughtExceptionHandler((t, e) -> System.err.println("异常：" + e))
      .stackSize(1024 * 1024 * 10) // 10 MB
      .start(() -> { });
```
