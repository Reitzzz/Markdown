## 1. 异常体系结构
Java 中所有异常和错误都继承自 `java.lang.Throwable`。

### 1.1 Throwable 层次
- `Throwable`
  - `Error`
  - `Exception`
    - `RuntimeException`
    - 其他受检异常

### 1.2 Error 错误
JVM 系统级问题（如 `OutOfMemoryError`、`StackOverflowError`），程序通常无法恢复，不建议捕获。

### 1.3 Exception 异常
程序运行中可处理的问题。
- **受检异常**：编译器强制处理（如 `IOException`、`SQLException`）。
- **非受检异常**：运行时异常（如 `NullPointerException`、`ArrayIndexOutOfBoundsException`），不强制处理。

## 2. 核心关键字

### 2.1 try-catch-finally
- `try`：包裹可能异常的代码。
- `catch`：捕获并处理异常。
- `finally`：无论是否异常都会执行。

```java
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("捕获异常");
} finally {
    System.out.println("执行完毕");
}
```

**多异常捕获**（Java 7+）：
```java
catch (IOException | SQLException e) { }
```

### 2.2 throw 与 throws
- `throw`：在方法体内抛出异常对象。
  ```java
  if (obj == null) throw new NullPointerException();
  ```
- `throws`：方法签名声明可能抛出的异常，交由调用者处理。
  ```java
  public void readFile() throws IOException { }
  ```

### 2.3 try-with-resources
Java 7 引入，自动关闭实现 `AutoCloseable` 的资源。

```java
try (FileInputStream fis = new FileInputStream("a.txt");
     FileOutputStream fos = new FileOutputStream("b.txt")) {
    // 操作流
} catch (IOException e) {
    // 处理异常
}
```
