## 1. 反射

### 1.1 概念
反射允许在运行时获取类的成员变量、成员方法和构造方法等信息，并可动态调用。

### 1.2 获取 Class 对象
- **源代码阶段**：`Class.forName("全限定类名")`
  ```java
  Class<?> clazz1 = Class.forName("com.example.Student");
  ```
- **加载阶段**：`类名.class`
  ```java
  Class<?> clazz2 = Student.class;
  ```
- **运行阶段**：`对象.getClass()`
  ```java
  Student stu = new Student();
  Class<?> clazz3 = stu.getClass();
  ```

### 1.3 公共解剖方法
- `getModifiers()`：获取修饰符。
- `getName()`：获取类名。
- `getParameters()`：获取形参对象数组。
- `getType()`：获取字段数据类型。

### 1.4 构造方法操作
| 方法 | 说明 |
|------|------|
| `getConstructors()` | 获取所有公共构造方法 |
| `getDeclaredConstructors()` | 获取所有构造方法 |
| `getConstructor(Class<?>...)` | 获取单个公共构造方法 |
| `getDeclaredConstructor(Class<?>...)` | 获取单个构造方法 |
| `setAccessible(true)` | 取消访问检查 |
| `newInstance(Object...)` | 创建实例 |

```java
Constructor<?> cons = clazz.getDeclaredConstructor(String.class, int.class);
cons.setAccessible(true);
Object obj = cons.newInstance("张三", 20);
```

### 1.5 成员变量操作
| 方法 | 说明 |
|------|------|
| `getFields()` | 获取所有公共成员变量 |
| `getDeclaredFields()` | 获取所有成员变量 |
| `getField(String)` | 获取单个公共成员变量 |
| `getDeclaredField(String)` | 获取单个成员变量 |
| `set(Object, Object)` | 设置值 |
| `get(Object)` | 获取值 |

```java
Field field = clazz.getDeclaredField("age");
field.setAccessible(true);
field.set(obj, 25);
Object value = field.get(obj);
```

### 1.6 成员方法操作
| 方法 | 说明 |
|------|------|
| `getMethods()` | 获取所有公共方法（含继承） |
| `getDeclaredMethods()` | 获取所有方法（不含继承） |
| `getMethod(String, Class<?>...)` | 获取单个公共方法 |
| `getDeclaredMethod(String, Class<?>...)` | 获取单个方法 |
| `invoke(Object, Object...)` | 调用方法 |

```java
Method method = clazz.getDeclaredMethod("study", String.class);
method.setAccessible(true);
Object result = method.invoke(obj, "Java");
```

### 1.7 反射进阶：类加载器与模块化

#### 双亲委派模型
- **Bootstrap ClassLoader**：加载核心类（`java.lang` 等），C++ 实现。
- **Platform ClassLoader**：加载扩展模块。
- **App ClassLoader**：加载 `classpath` 下的自定义类。

**工作流程**：自底向上检查是否已加载，自顶向下尝试加载。  
**目的**：防止核心类被篡改。

**破坏双亲委派的场景**：SPI 机制（如 JDBC）、Tomcat 类隔离。

#### Java 9+ 模块化对反射的影响
默认无法通过 `setAccessible(true)` 访问模块内非 `public` 成员。  
解决方案：
1. 启动参数：`--add-opens java.base/java.util=ALL-UNNAMED`
2. 模块声明：`open module` 或 `opens` 包名。

#### 自定义类加载器
继承 `ClassLoader`，重写 `findClass` 方法：
```java
class MyClassLoader extends ClassLoader {
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        byte[] data = loadClassData(name); // 读取 .class 文件
        return defineClass(name, data, 0, data.length);
    }
}
```

## 2. 泛型

### 2.1 引入
解决类型安全与强制转换问题，避免 `Object` 带来的类型不确定性。

### 2.2 使用

- **泛型类**
  ```java
  public class GenericBox<T> {
      private T content;
      public T getContent() { return content; }
      public void setContent(T content) { this.content = content; }
  }
  GenericBox<Apple> box = new GenericBox<>();
  box.setContent(new Apple());
  Apple content = box.getContent();
  ```

- **泛型接口**
  ```java
  public interface Generator<T> {
      T next();
  }
  public class RandomGenerator implements Generator<String> {
      public String next() { return "Random String"; }
  }
  ```

- **泛型方法**
  ```java
  public static <E> void printArray(E[] array) {
      for (E e : array) System.out.println(e);
  }
  ```

### 2.3 通配符

- **上界通配符 `? extends T`**：表示 T 或 T 的子类，**只能读取，不能写入**。
- **下界通配符 `? super T`**：表示 T 或 T 的父类，**只能写入 T 类型，读取受限**。
- **无界通配符 `?`**：未知类型，只关心容器本身属性（如大小）。

### 2.4 泛型进阶：类型擦除与 PECS 原则

#### 类型擦除机制
泛型仅存在于编译期，字节码中无泛型信息。
- `List<T>` → `List`（Object）
- `List<T extends Number>` → `List`（Number）

**擦除限制**：不能 `new T()`、不能 `instanceof` 泛型参数、静态变量不能用泛型参数。

#### PECS 原则
- **Producer Extends（PE）**：提供数据，使用 `<? extends T>`，只读。
- **Consumer Super（CS）**：消费数据，使用 `<? super T>`，只写。

```java
// PE：读取
public void printList(List<? extends Fruit> list) {
    for (Fruit f : list) System.out.println(f);
    // list.add(new Apple()); // 编译错误
}

// CS：写入
public void addApple(List<? super Apple> list) {
    list.add(new Apple());
    // Fruit f = list.get(0); // 编译错误
}
```

## 3. 注解

### 3.1 作用
不是程序本身，但对程序作出解释，可被其他程序读取（如编译器、框架）。

### 3.2 使用
格式：`@注解名(参数名=值)`。可附加在包、类、方法、字段等位置。

### 3.3 内置注解
- `@Override`：标记重写方法。
- `@Deprecated`：标记已过时。
- `@SuppressWarnings`：抑制警告（如 `"unchecked"`、`"all"`）。

### 3.4 元注解
- `@Target`：指定适用范围（`ElementType.TYPE`、`METHOD`、`FIELD` 等）。
- `@Retention`：指定保留策略（`SOURCE`、`CLASS`、`RUNTIME`）。
- `@Documented`：包含在 javadoc 中。
- `@Inherited`：允许子类继承注解。

### 3.5 自定义注解
```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@interface MyAnnotation {
    String name() default "";
    int age() default 0;
    int[] arr() default {};
}

public class Demo {
    @MyAnnotation(name = "lc", age = 18)
    public void method() { }
}
```
