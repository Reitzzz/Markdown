## 异常体系结构

Java中所有的异常和错误都继承于 `java.lang.Throwable` 类。

### Error错误
通常是JVM系统级别的问题，如内存溢出 (`OutOfMemoryError`) 或栈溢出 (`StackOverflowError`)。程序通常无法恢复，不建议代码去catch它。

### Exception
程序运行中产生的可以处理的问题。

- 受检异常
编译器强制要求处理的异常，不处理，代码无法编译通过。如 `IOException`，`SQLException`。

- 非受检异常
即运行时异常，不强制处理。如 `NullPointerException`，`ArrayIndexOutOfBoundsException`。

## 核心关键字

### try
包裹可能出现异常的代码块。必须配合catch或finally。

### catch
捕获异常并定义出现问题后该怎么办。

### finally
无论是否有异常发生，一定会被执行的代码块。

```java
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Exception caught");
} finally {
    System.out.println("Execution finished");
}
```

### throw
动作，在方法体内内部主动抛出一个异常对象，用来主动中断流程。

```java
if (obj == null) {
    throw new NullPointerException();
}
```

### throws
声明。用在方法签名上告诉调用者本方法可能会抛出这些异常，请调用者自行处理。

```java
public void readFile() throws IOException {
}
```
