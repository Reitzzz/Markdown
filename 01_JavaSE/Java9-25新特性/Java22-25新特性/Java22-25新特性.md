## 1.1 新一代 LTS 版本：Java 25 概述

距离 Java 21 发布过去两年后，按照 Oracle 每两年发布一个长期支持版本（LTS）的承诺，在 2025 年 9 月，Java 迎来了全新的 LTS 版本——**Java 25**。

Java 22 到 24 作为过渡版本，不断地孵化和打磨新功能。而在 Java 25 中，许多经历了多次预览的重磅功能终于“转正”，成为了可以放心在企业级生产环境中使用的最终特性。接下来我们就挑选几个比较实用、且能直接改善我们日常编码体验的新特性进行讲解。

## 1.2 无名变量与模式

*(Java 22 转正)*

在开发中，我们经常遇到这种情况：为了符合语法规范，我们必须声明一个变量（比如在 `catch` 块中，或者在 `for` 循环、Lambda 表达式中），但实际上后面的代码根本不需要使用这个变量。这不仅会触发编译器的未使用警告，看起来也不够清晰。

Java 22 正式引入了“无名变量”，我们可以直接用下划线 `_` 来代替那些不需要使用的变量：

```java
// 以前在处理异常时，哪怕不需要用到 exception 对象，也得给个变量名
try {
    int number = Integer.parseInt("Not a Number");
} catch (NumberFormatException e) {   // 这里的 e 根本没用到
    System.out.println("转换失败！");
}

// Java 22 之后，可以直接用 _ 丢弃
try {
    int number = Integer.parseInt("Not a Number");
} catch (NumberFormatException _) {   // 使用 _ 表示我不关心具体的异常对象
    System.out.println("转换失败！");
}
```

这不仅适用于 `catch` 块，在普通的循环中同样好用：

```java
// 比如我们只想循环 5 次执行某个动作，并不关心索引值
for (int _ = 0; _ < 5; _++) {
    System.out.println("Hello World!");
}
```

## 1.3 灵活的构造器主体

*(Java 25 转正)*

在以前的 Java 版本中，如果你要在构造方法里显式调用父类的构造方法（`super(...)`）或者是自己的其他构造方法（`this(...)`），这行代码**必须**放在当前构造器的第一行。

这个限制非常死板，因为有时候我们希望在实例化父类之前，先对传进来的参数进行一下验证或者预处理逻辑。

现在在Java25，**允许先进行其他处理，再调用构造方法**。

```java
public class Customer extends Person {
    final String email;

    public Customer(String email) {
        // Java 25 之前：必须先把 super() 放在第一行，很难在它之前进行格式校验
        // super(); 
        
        // Java 25 之后：允许在 super() 之前编写逻辑代码
        if (!email.contains("@")) {
            throw new IllegalArgumentException("邮箱格式不正确");
        }
        
        super(); // 校验通过后，再调用父类构造器完成实例化
        this.email = email;
    }
}
```

这个特性极大地提升了我们编写带参构造方法时的灵活性，消除了不必要的静态辅助方法。

## 1.4 Markdown 文档注释

*(Java 23 转正)*

如果你经常写 API 文档，一定会对传统的 JavaDoc 注释感到痛苦——满屏幕的 HTML 标签（像 `<p>`, `<ul>`, `<pre>`），写起来繁琐，在代码里直接阅读时更是杂乱无章。

Java 23 终于做出了改变，官方正式支持使用 **Markdown** 来编写 JavaDoc 注释！它的语法前缀由传统的 `/**` 变为了三个斜杠 `///` ：

```java
/// # 这是一个测试类
/// 该类展示了如何使用全新的 Markdown 语法编写注释。
/// 
/// 我们可以很方便地插入代码块，再也不用手写 <pre> 标签了：
/// ```java
/// public static void main(String[] args) {
///     System.out.println("Hello Markdown!");
/// }
/// ```
/// 并且依然可以完美兼容 `@param` 和 `@return` 等传统文档标签：
/// @param name 传入的名字
public void hello(String name) {
    // ...
}
```

相较于旧版的 HTML 标签混编，Markdown 格式无疑让代码注释变得更加易读和易维护。

## 1.5 作用域值

*(Java 25 转正)*

在多线程开发中，如果我们想在同一个线程的上下文中传递一些全局共享的数据（比如用户登录凭证、事务 ID），通常会使用 `ThreadLocal`。但是 `ThreadLocal` 存在一些缺陷：它的数据是可变的、如果忘记清理容易造成内存泄漏，并且在配合海量的虚拟线程（Virtual Threads）使用时，内存开销可能会非常大。

Java 25 正式推出了 **Scoped Values（作用域值）** 作为现代化的替代方案。它是不可变的，并且仅在限定的代码块（作用域）内有效：

```java
public class Server {
    // 定义一个静态的作用域值
    private static final ScopedValue<String> USER_CONTEXT = ScopedValue.newInstance();

    public void serve(Request request) {
        String username = "Admin"; // 假设从请求中解析出的用户名
        
        // 只有在这个 run() 包含的代码块以及它调用的方法内部，USER_CONTEXT 才绑定为 "Admin"
        // 一旦超出这个范围，绑定自动失效，无需手动清理，从根本上杜绝了内存泄漏
        ScopedValue.where(USER_CONTEXT, username).run(() -> {
            doSomething(); 
        });
    }

    public void doSomething() {
        // 在内部随时可以获取绑定的值
        String current = USER_CONTEXT.get();
        System.out.println("当前处理用户: " + current);
    }
}
```

## 1.6 模块导入声明增强

当我们编写简单的 Java 程序时，往往需要在文件头部写上一长串的 `import` 语句（比如导入各种集合类、文件流、并发包等）。为了简化这类操作，Java 引入了模块级别的一键导入：

```java
// 以前我们需要一行行导入所需的类
// import java.util.List;
// import java.util.Map;
// import java.util.stream.Collectors;

// 现在，可以直接一行代码导入整个 java.base 模块暴露出来的所有包！
//相当于#include <bits/stdc++.h>

import module java.base;

public class Main {
    public static void main(String[] args) {
        // 无缝使用 List, Map, Stream 等核心基础类，文件头部变得极其清爽
        List<String> list = List.of("Java", "25", "LTS");
        System.out.println(list);
    }
}
```

这个功能在日常编写小工具类、刷算法题，或者配合 JShell 使用时，能够省去大量繁琐的导包工作。
