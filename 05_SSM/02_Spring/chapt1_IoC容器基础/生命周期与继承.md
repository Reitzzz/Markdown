# Spring Bean 生命周期与继承学习笔记

## 1. Bean 的生命周期管理
除了修改构造方法，我们可以为 Bean 指定初始化方法和销毁方法，以便在对象创建和被销毁时执行特定的任务。

### 1.1 定义初始化与销毁方法
在 Bean 对应的类中，编写普通的方法作为初始化和销毁的回调方法：

```java
public void init() {
    System.out.println("我是对象初始化时要做的事情！");    
}

public void destroy() {
    System.out.println("我是对象销毁时要做的事情！");
}
```

在 XML 配置文件中，通过 `init-method` 和 `destroy-method` 属性来指定这两个方法：

```xml
<bean name="student" class="com.test.bean.Student" init-method="init" destroy-method="destroy"/>
```

### 1.2 生命周期方法的执行时机
Bean 的作用域（Scope）会直接影响其生命周期方法的执行：

```java
// 当容器创建时，默认情况下Bean都是单例的，都会在一开始就加载好。对象构造完成后，会执行init-method
ClassPathXmlApplicationContext context = new ClassPathXmlApplicationContext("test.xml");

// 可以调用close方法关闭容器，此时容器内存放的单例Bean也会被一起销毁，并执行destroy-method
context.close();
```

* **单例模式 (Singleton，默认)**：容器启动时创建对象并调用 `init-method`。容器关闭时销毁对象并调用 `destroy-method`。Spring 能够管理其完整的生命周期。
* **原型模式 (Prototype)**：只在每次获取 Bean 时才创建对象并调用 `init-method`。**对应的销毁方法不会被调用**。

> **Spring 官方文档说明：**
> In contrast to the other scopes, Spring does not manage the complete lifecycle of a prototype bean. The container instantiates, configures, and otherwise assembles a prototype object and hands it to the client, with no further record of that prototype instance. Thus, although initialization lifecycle callback methods are called on all objects regardless of scope, in the case of prototypes, configured destruction lifecycle callbacks are not called. The client code must clean up prototype-scoped objects and release expensive resources that the prototype beans hold.

## 2. Bean 的属性继承
Spring 中的 Bean 继承指的是**配置属性的继承**，而不是 Java 类的继承。当多个 Bean 具有大量相同的属性注入时，可以使用继承来简化配置。

### 2.1 基础实现
假设有两个类 `SportStudent` 和 `ArtStudent`，它们都具有 `name` 属性：

```java
public class SportStudent {
    private String name;
    public void setName(String name) { this.name = name; }
}

public class ArtStudent {
    private String name;
    public void setName(String name) { this.name = name; }
}
```

正常情况下需要分别为它们配置属性注入：

```xml
<bean name="artStudent" class="com.test.bean.ArtStudent">
    <property name="name" value="小明"/>
</bean>

<bean class="com.test.bean.SportStudent">
    <property name="name" value="小明"/>
</bean>
```

使用 `parent` 属性后，`SportStudent` 可以直接继承 `artStudent` 配置的属性：

```xml
<bean name="artStudent" class="com.test.bean.ArtStudent">
    <property name="name" value="小明"/>
</bean>

<bean class="com.test.bean.SportStudent" parent="artStudent"/>
```
*注意：父 Bean 中配置的所有属性，在子 Bean 的类中必须同样存在且提供相应的 setter 方法，否则会报错。*

### 2.2 属性覆盖与抽象模板 Bean
子 Bean 在继承父 Bean 属性的基础上，可以对其进行重写或增加新的属性配置：**子 Bean 自己显式配置的属性优先级高于从父 Bean 继承来的属性**

```xml
<bean name="artStudent" class="com.test.bean.ArtStudent">
    <property name="name" value="小明"/>
    <property name="id" value="1"/>
</bean>

<bean class="com.test.bean.SportStudent" parent="artStudent">
    <property name="id" value="2"/>
</bean>
```

如果某个 Bean 仅仅是为了作为配置模板供其他 Bean 继承，并不需要被实例化，可以添加 `abstract="true"` 属性将其声明为抽象 Bean：

```xml
<bean name="artStudent" class="com.test.bean.ArtStudent" abstract="true">
    <property name="name" value="小明"/>
</bean>

<bean class="com.test.bean.SportStudent" parent="artStudent"/>
```
*注意：一旦声明为抽象 Bean，IoC 容器就不会创建该 Bean 的对象，也无法通过容器获取到它的实例。*

## 3. 全局默认配置
如果希望整个 Spring 上下文中所有的 Bean 都采用某种统一的配置规则，可以在最外层的 `<beans>` 标签中进行默认配置。

例如（概念示例）：
```xml
<beans default-autowire="byType" default-lazy-init="true" ...>
    </beans>
```
除非 Bean 内部显式进行了特定配置以覆盖默认值，否则最外层编写的全局默认配置对所有 Bean 生效。
