# 注解开发与整合技术速查手册

本手册按照实际项目开发中从基础配置、面向切面编程、事务管理到数据库整合的使用顺序列出了相关注解。

## 1. 核心容器与Bean管理 (IoC/DI)

### 1.1 基础配置与组件扫描
* **`@Configuration`**: 用于声明一个配置类，替代传统的XML配置文件。
* **`@ComponentScan`**: 用于在配置类中开启包扫描，指示Spring查找Bean的路径。
* **`@Component`**: 添加在目标类上，将其快速注册为Spring容器内的Bean。默认命名规则为类名首字母小写。
* **`@Import`**: 可以在当前配置类中引入其他的配置类 **(类似于嵌套)**。

```java
@Configuration
@ComponentScan("com.test.bean")
@Import(LBWConfiguration.class)
public class MainConfiguration {
}
```

### 1.2 Bean的注册与属性控制
* **`@Bean`**: 用于方法上方，用于配置Bean类的具体属性。支持配置`name`、`initMethod`、`destroyMethod`、`autowireCandidate`等属性。
* **`@Scope`**: 用于配置Bean的作用域，例如配置为`"prototype"`来使用原型/多例模式, 配置为 "singleton"来使用单例模式。
* **`@Lazy`**: 配置Bean是否懒加载，对应XML中的`lazy-init`属性。
* **`@DependsOn`**: 配置Bean的创建依赖顺序，对应XML中的`depends-on`属性。

```java
@Bean(name = "student", initMethod = "init", destroyMethod = "destroy")
@Lazy(true)
@Scope("prototype")
@DependsOn("teacher")
public Student student(){
    return new Student();
}
```

### 1.3 依赖注入 (DI)
* **`@Autowired`**: 由IoC容器自动完成依赖注入，可用于字段、构造方法或Setter方法，默认采用byType方式装配。
* **`@Qualifier`**: 当出现多个相同类型的Bean时，配合`@Autowired`使用，通过指定名称进行精确匹配查找。
* **`@Resource`**: Jakarta EE规范注解，作用类似`@Autowired`，但默认采用byName方式装配，找不到时再通过byType寻找，可用于字段和Setter方法。

```java
public class Student {
    @Autowired
    @Qualifier("a")
    private Teacher teacher;
}
```

### 1.4 生命周期控制
* **`@PostConstruct`**: 标识Bean的初始化方法，效果等同于配置`init-method`。
* **`@PreDestroy`**: 标识Bean的销毁方法，效果等同于配置`destroy-method`。

```java
@PostConstruct
public void init(){
}

@PreDestroy
public void destroy(){
}
```

## 2. 面向切面编程 (AOP)

### 2.1 开启与切面定义
* **`@EnableAspectJAutoProxy`**: 添加在主配置类上，用于开启AOP注解支持。
* **`@Aspect`**: 声明当前类是一个切面类，通常配合`@Component`一起将其注册为Bean。

### 2.2 通知行为 (Advice)
* **`@Before`**: 前置通知，在切点指定的被增强方法执行之前触发。支持命名绑定模式获取参数。
* **`@AfterReturning`**: 返回后通知，在目标方法成功执行并返回后触发，可以通过`returning`属性接收被增强方法的返回值。
* **`@Around`**: 环绕通知，包裹目标方法的执行，通过调用`ProceedingJoinPoint.proceed()`来主动执行目标方法，可控制执行前后的逻辑。

```java
@Aspect
@Component
public class StudentAOP {

    @Before("execution(* org.example.entity.Student.study())")
    public void before(JoinPoint point){
    }

    @Around("execution(* com.test.bean.Student.test(..))")
    public Object around(ProceedingJoinPoint point) throws Throwable {
        return point.proceed();
    }
}
```

## 3. 事务管理 (TX)

### 3.1 开启与配置
* **`@EnableTransactionManagement`**: 标注在Spring主配置类上，开启声明式事务的AOP支持。
* **`@Transactional`**: 声明事务行为，可标注在方法或类上，保证内部操作处于同一事务中，异常时自动回滚。可配置传播规则(`propagation`)、隔离级别(`isolation`)、超时时间(`timeout`)、只读属性(`readOnly`)及回滚规则(`rollbackFor`)。

```java
@Transactional(propagation = Propagation.REQUIRED, rollbackFor = Exception.class)
public void test() {
}
```

## 4. MyBatis与数据库整合

### 4.1 自动扫描与注册
* **`@MapperScan`**: 添加在配置类上，自动扫描指定包下所有接口，将其注册为MyBatis的Mapper Bean。
* **`@Mapper`**: 标注在接口上，标识该接口为MyBatis的Mapper组件(即SQL仙人指路)。

### 4.2 基础SQL操作
* **`@Select`**: 在接口方法上编写并绑定SQL查询语句。
* **`@Insert`**: 在接口方法上编写并绑定SQL插入语句。

```java
@Mapper
public interface TestMapper {
    @Select("select * from student where sid = 1")
    Student getStudent();

    @Insert("insert into student(name, sex) values('测试', '男')")
    void insertStudent();
}
```

## 5. 辅助工具扩展 (Lombok)

* **`@Data`**: Lombok框架提供的注解，用于自动为实体类生成Getter/Setter方法、`toString()`等基础代码。
* **`@Slf4j`**: Lombok框架提供的注解，为当前类自动注入SLF4J规范的日志对象，免去手动实例化Logger的过程。

```java
@Data
public class Student {
    private int sid;
    private String name;
    private String sex;
}
```
